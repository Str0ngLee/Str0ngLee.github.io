Windows Docker 部署 OpenList 挂载网盘完整指南：夸克网盘 + 一刻相册踩坑全记录
本文记录了在 Windows 11 环境下，基于 Docker Desktop 部署 OpenList（AList 社区分支），挂载夸克网盘、一刻相册，并通过 RaiDrive 映射为本地盘符的完整流程。文中整理了部署过程中遇到的所有问题与对应解决方案，适合想要统一管理多网盘资源的朋友参考。
运行环境：Windows 11 | Docker Desktop | Edge 浏览器
⚠️ 快速找回入口：可在浏览器扩展管理页找到 Cookie 提取工具，或直接打开本地数据目录下的 README 备忘文档。

二、Docker 部署步骤

  1. 拉取镜像并启动容器
    docker pull openlistteam/openlist:latest
    docker run -d
    –restart=always
    -v D:/your-openlist-data:/opt/openlist/data
    -p 15244:5244
    –name=”openlist”
    openlistteam/openlist:latest
  • 端口说明：OpenList 默认端口 5244 在 Windows 上可能被系统保留，因此映射为本地 15244 端口
  • 数据持久化：将容器内数据目录挂载到本地 D:/your-openlist-data，替换为你自己的本地目录即可
  1. 首次登录获取初始密码
    容器首次启动会在日志中生成管理员初始密码，执行以下命令查看：
    docker logs openlist | tail -10
    在日志中找到 Successfully created the admin user and the initial password is: xxxxxxxx 即可获取初始密码，登录后建议自行修改。

三、夸克网盘挂载配置
痛点：手动复制 Cookie 流程繁琐
夸克网盘需要 Cookie 认证，通过浏览器 F12 手动查找复制步骤多、效率低。
解决方案：Edge 扩展一键提取 Cookie
扩展安装：

  1. 打开 Edge 浏览器，地址栏输入 edge://extensions
  2. 开启页面右上角「开发人员模式」
  3. 点击「加载解压缩的扩展」，选择本地数据目录下的 quark_cookie_ext 文件夹
    使用方法：
  4. 在 Edge 中登录夸克网盘官网 https://pan.quark.cn/
  5. 点击浏览器扩展栏的 Cookie 提取工具，选择「夸克网盘」，点击「提取 Cookie」后复制内容
  6. 进入 OpenList 管理后台 → 存储 → 添加存储，驱动选择「夸克」，粘贴 Cookie，挂载路径填写 /quark，保存即可
    注意事项
  • 夸克 Cookie 存在有效期（通常几天到几周），过期后重新执行提取粘贴流程即可
  • 认证需要包含所有子域的 Cookie，扩展已自动处理跨子域提取，无需手动逐个复制

四、一刻相册挂载与踩坑
问题 1：添加后页面持续转圈加载失败
现象： 添加存储保存后，进入目录页面一直转圈，浏览器提示页面无响应。
根因： Docker 容器内 DNS 解析一刻相册相关域名（百度 CDN）耗时过长，导致 API 请求超时被取消。夸克网盘无此问题是因为服务器解析更快。
排查过程：

  1. 最初怀疑 Cookie 不完整，优化了跨子域提取逻辑，未解决
  2. 修改 Docker DNS 为国内公共 DNS，仍然超时
  3. 硬编码域名解析 IP 后问题彻底解决
    最终解决命令：

停止并删除原有容器

docker stop openlist && docker rm openlist

重建容器，添加域名解析与国内 DNS

docker run -d –restart=always
-v D:/your-openlist-data:/opt/openlist/data
-p 15244:5244
–add-host=”photo.baidu.com:119.249.103.51”
–dns 223.5.5.5
–name=”openlist”
openlistteam/openlist:latest
问题 2：大量图片导致浏览器卡死
现象： 网络问题解决后页面可加载，但浏览图片目录时浏览器逐渐无响应。
根因： 后端 API 支持分页，但前端会批量渲染缩略图，数万张图片的渲染量超出浏览器负载。
两种解决方案：

  • 方案 A（推荐）： 通过 RaiDrive 以 WebDAV 挂载到本地盘符，用系统资源管理器浏览，无渲染压力
  • 方案 B： 浏览器访问时手动添加分页参数，例如 http://localhost:15244/yike?per_page=30

五、RaiDrive 挂载 WebDAV 配置
问题：挂载报错 protocolVersion / 无法连接
现象： 使用 Windows 自带「添加网络位置」或 RaiDrive 挂载均失败，提示协议版本错误或无法打开路径。
根因： Windows 原生 WebDAV 客户端与 RaiDrive 默认走 HTTPS 协议，但本地部署的 OpenList 仅开启 HTTP 服务，未配置 SSL 证书。
正确配置方式
添加存储时取消勾选「安全连接」（地址前的锁形图标），强制走 HTTP 协议，具体参数如下：

安全说明

  • 本地部署服务仅监听 localhost，未暴露公网，无外部访问风险
  • 未配置 UPnP 与端口映射规则，服务仅本机可访问
  • 保持 TLS 证书验证开启，避免中间人风险
  • 使用版本 v4.2.2，高于修复高危 CVE 的 v4.1.10 版本
